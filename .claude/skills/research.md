# Research Skill

## Overview
Systematic research and investigation of technologies, patterns, and best practices to inform project decisions.

## Research Process

### Phase 1: Define the Question
- Clearly articulate what needs to be researched
- Identify the problem or decision to be made
- Set scope boundaries
- List known constraints

Example:
- "How to implement real-time notifications without polling?"
- "What's the best way to handle form state in React?"
- "How do we optimize bundle size for mobile users?"

---

### Phase 2: Gather Information
- Search documentation and official guides
- Review community best practices
- Study relevant articles and blog posts
- Examine similar implementations
- Check GitHub discussions and issues
- Review Stack Overflow discussions
- Look at industry standards

---

### Phase 3: Analyze Options
Create comparison matrix:
| Aspect | Option A | Option B | Option C |
|--------|----------|----------|----------|
| Ease of learning | ★★☆☆☆ | ★★★★☆ | ★★★☆☆ |
| Performance | ★★★★★ | ★★★☆☆ | ★★★★☆ |
| Community support | ★★★★☆ | ★★★★★ | ★★☆☆☆ |
| Maintenance | ★★★☆☆ | ★★★★☆ | ★★★☆☆ |

---

### Phase 4: Make Recommendation
- Recommend best approach for project needs
- Explain trade-offs
- List pros and cons
- Suggest next steps

---
