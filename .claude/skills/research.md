# Research Skill

## Process

1. **Define Question** - Clarify what needs research, set scope
2. **Gather Info** - Search docs, articles, GitHub discussions, StackOverflow
3. **Analyze** - Compare options with trade-offs
4. **Recommend** - Choose best approach for project needs
5. **Document** - Create implementation guide

## Research Topics

**React/Ecosystem**: State management, forms, data fetching, routing, animation, component libraries

**TypeScript**: Advanced types, conditional types, mapped types, generics

**Performance**: Bundle optimization, image optimization, code splitting, caching, SSR

**Tooling**: Build tools (Vite, Webpack), testing (Vitest, Jest, Cypress), linting (ESLint)

**Architecture**: Micro frontends, module federation, design systems, monorepos

**Security**: Auth methods, API security, secure storage, compliance

**Accessibility**: WCAG standards, accessible components, screen readers, testing

## Output Format
```markdown
# Research: [Topic]

**Question**: [What we're researching and why]

**Findings**:
- Option 1: [pros/cons]
- Option 2: [pros/cons]

**Recommendation**: [Best choice with rationale]

**Implementation**: [Quick steps or resources]

**Resources**: [Links to documentation/articles]
```
